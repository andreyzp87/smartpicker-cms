# Export Data Contract

This document describes the JSON files generated by SmartPicker CMS for the Astro frontend.

## Overview

Exports are written to `data/exports/` and served from `/api/exports/*`.

The contract is optimized for static generation:

- `products.json` is the lightweight global device index
- `products/slugs.json` drives `/devices/[slug].astro`
- `products/{slug}.json` contains full device details
- manufacturer, hub, category, and protocol pages get their own exports
- `site.json` powers the homepage
- `sitemap.json` powers Astro sitemap generation
- `types.ts` exports the TypeScript contract for the Astro repo

## Contract Version

Current version: `3.0`

Every JSON file includes:

```json
{
  "generated": "2026-03-12T18:24:50.373Z",
  "version": "3.0"
}
```

## Export Rules

- Only products with `status = published` are exported.
- `products.json` contains summary records only.
- `products/{slug}.json` contains full detail data for a single product.
- `manufacturers.json`, `hubs.json`, and `categories.json` only include entities with at least one published product.
- `protocols.json` always includes all supported protocol pages, even when a protocol currently has zero published products.
- `categories/paths.json` is the source of truth for category `getStaticPaths()`.
- `sitemap.json` contains public site routes, not CMS export URLs.

## File Layout

```text
/api/exports/
  products.json
  products/slugs.json
  products/{slug}.json
  manufacturers.json
  manufacturers/slugs.json
  manufacturers/{slug}.json
  hubs.json
  hubs/slugs.json
  hubs/{slug}.json
  categories.json
  categories/paths.json
  categories/{path}.json
  protocols.json
  protocols/slugs.json
  protocols/{slug}.json
  site.json
  sitemap.json
  types.ts
```

## Products

### `products.json`

Purpose:

- `/devices`
- client-side filtering
- search index bootstrapping
- joining device data into hub/manufacturer/category/protocol pages

Type:

```ts
export interface ProductExportSummary {
  id: number
  slug: string
  name: string
  manufacturer: ExportEntityRef | null
  model: string | null
  category: ExportEntityRef | null
  categoryPath: string[]
  primaryProtocol: string | null
  localControl: boolean | null
  cloudDependent: boolean | null
  requiresHub: boolean | null
  matterCertified: boolean | null
  imageUrl: string | null
  compatibleHubSlugs: string[]
  compatibilityStatuses: string[]
  updatedAt: string
  searchText: string
}

export interface ProductsIndexResponse {
  generated: string
  version: string
  count: number
  products: ProductExportSummary[]
}
```

Example:

```json
{
  "generated": "2026-03-12T18:24:50.373Z",
  "version": "3.0",
  "count": 5,
  "products": [
    {
      "id": 1,
      "slug": "2-green-homes-zbr1",
      "name": "ZBR1 - Zigbee board for Raspberry Pi",
      "manufacturer": {
        "id": 73,
        "slug": "2-green-homes",
        "name": "2 Green Homes"
      },
      "model": "ZBR1",
      "category": null,
      "categoryPath": [],
      "primaryProtocol": "zigbee",
      "localControl": null,
      "cloudDependent": null,
```

Purpose:

## Recommended Astro Data Helpers

```ts
const CMS_URL = import.meta.env.CMS_URL || 'https://cms.smartpicker.io'

export async function getProductsIndex() {
  const res = await fetch(`${CMS_URL}/api/exports/products.json`)
  if (!res.ok) throw new Error('Failed to fetch products index')
  return res.json()
}

export async function getProductSlugs() {
  const res = await fetch(`${CMS_URL}/api/exports/products/slugs.json`)
  if (!res.ok) throw new Error('Failed to fetch product slugs')
  return res.json()
}

export async function getProduct(slug: string) {
  const res = await fetch(`${CMS_URL}/api/exports/products/${slug}.json`)
  if (!res.ok) return null
  return res.json()
}

export async function getManufacturerSlugs() {
  const res = await fetch(`${CMS_URL}/api/exports/manufacturers/slugs.json`)
  if (!res.ok) throw new Error('Failed to fetch manufacturer slugs')
  return res.json()
}

export async function getCategoryPaths() {
  const res = await fetch(`${CMS_URL}/api/exports/categories/paths.json`)
  if (!res.ok) throw new Error('Failed to fetch category paths')
  return res.json()
}
```

## Recommended Astro Page Patterns

### `/devices/[slug].astro`

```astro
---
import { getProduct, getProductSlugs } from '../../lib/data'

export async function getStaticPaths() {
  const { slugs } = await getProductSlugs()
  return slugs.map((slug: string) => ({ params: { slug } }))
}

const data = await getProduct(Astro.params.slug!)
if (!data) return Astro.redirect('/404')

const { product } = data
---
```

### `/categories/[...slug].astro`

```astro
---
import { getCategoryPaths } from '../../lib/data'

export async function getStaticPaths() {
  const { categories } = await getCategoryPaths()

  return categories.map((category: { pathSegments: string[] }) => ({
    params: { slug: category.pathSegments },
  }))
}
---
```

## Notes

- Use `products.json` for filtering and global joins.
- Use per-entity detail files for page-specific metadata and slug lists.
- Prefer `deviceSlugs` over embedded device objects to avoid duplicated payloads.
- `searchText` is already normalized for lightweight client-side search.
